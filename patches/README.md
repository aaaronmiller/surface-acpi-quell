# Patches — Upstream Submission Guide

This directory contains kernel patches generated by the Surface ACPI Quell project.
They are starting points for upstream submission, not final-form patches.

## Patch 1: ACPI CA — Suppress repeated "no handler" errors

**File:** `0001-ACPI-CA-Suppress-repeated-no-handler-errors-for-disa.patch`

**Target:** ACPICA project (upstream of the ACPI Component Architecture used by Linux)

**Submission path:**
1. File a bug/report at https://acpica.org/community
2. The ACPICA maintainers import patches into their tree
3. Linux kernel picks them up from the ACPICA import

**Do NOT submit directly to LKML** — this code is imported from the ACPICA project.
Patches to ACPICA code must go through their process first.

## Patch 2: Surface ACPI quirk — mask unhandled GPEs

**File:** `0002-ACPI-Surface-Add-quirk-to-mask-unhandled-GPEs-on-Mic.patch`

**Target:** linux-surface kernel fork (https://github.com/linux-surface/linux-surface)

**Submission path:**
1. Open a PR at https://github.com/linux-surface/linux-surface
2. The linux-surface maintainers carry Surface-specific patches
3. After proving stable there, submit to LKML for mainline inclusion

**Note:** The quirk currently adds a new CONFIG_SURFACE_ACPI_QUIRK symbol.
Before mainline submission, this must be changed to use the existing
CONFIG_ACPI_QUIRKS infrastructure to avoid adding new config symbols.

## Patch 3: Our kernel module (surface_fixed_event_quell.ko)

**Not for upstream submission.** This module masks IRQ9 (ACPI SCI), which is
too aggressive for general consumption. It's a workaround for a firmware bug
on Microsoft Surface devices where the Surface Aggregator Module handles
hardware monitoring instead of ACPI.

## Requirements for Upstream

Before any patch can be accepted:

1. **Signed-off-by:** Every patch must have a `Signed-off-by:` line from a
   real human who can defend the code
2. **Coding style:** Run `scripts/checkpatch.pl` on each patch and fix all warnings
3. **Tested on:** At least 2 different Surface models
4. **No new Kconfig symbols:** Use existing infrastructure (CONFIG_ACPI_QUIRKS)
5. **Reviewed by:** At least one other kernel developer
